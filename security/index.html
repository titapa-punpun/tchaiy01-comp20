Location

gamemanager.js: line 92

Severity of Issue

Medium Severity. Although this is technically not exactly a vulnerability, it is really bad programming practice because hard-coding username is extremely rigid and constrained.

Description of Issue

Hard-coding username is very inflexible because the username will always be "Ethan" in this case. It doesn't allow user to input his/her actual username because it will always default to the hard-coded username. Users are unable to save their scores to a unique name, which means it will be hard to identify which score belongs to which user since they are all "Ethan."
